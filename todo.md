## 🐛 已知问题与瑕疵

### P0 - 必须修复

#### 1. 实时更新机制缺陷 🔴
- **位置**: `src/App.tsx:61-66`
- **问题描述**: 
  - 实时文件/目录发现事件（fileFound/directoryFound）已被禁用
  - 改为扫描完成时一次性返回所有结果
  - 根本原因：前端渲染性能不足以支持大量实时更新
- **影响**: 用户无法看到扫描过程中的中间结果，大目录扫描时体验较差
- **修复建议**: 
  - 重新启用增量更新（每 N 个结果推送一次，而非逐个推送）
  - 配合虚拟滚动（react-virtual 或 @tanstack/react-virtual）优化渲染性能
  - 参考实现：在 scanner.rs 中使用节流机制，每 100-500 个文件发送一次事件

#### 2. 取消扫描逻辑存在 Workaround 🔴
- **位置**: `src/App.tsx:187-200`
- **问题描述**:
  ```typescript
  setTimeout(() => {
    // 5秒超时强制重置状态
  }, 5000)
  ```
  - 这段代码表明事件同步机制存在问题
  - 取消事件可能未能正确、及时地触发状态更新
- **影响**: 取消操作后状态可能不一致，需要强制重置
- **修复建议**:
  - 审查 Tauri 事件发送/接收链路
  - 确保 cancel_scan 命令执行后立即发送 Cancelled 事件
  - 考虑添加重试机制或状态确认机制
  - 移除 workaround 代码

#### 3. 删除操作同步阻塞 🔴
- **位置**: `src-tauri/src/commands.rs:196-199`
- **问题描述**:
  ```rust
  if path_buf.is_dir() {
      std::fs::remove_dir_all(&path_buf)  // 同步阻塞！
  } else {
      std::fs::remove_file(&path_buf)     // 同步阻塞！
  }
  ```
  - 大目录删除会长时间阻塞 UI 线程
- **影响**: 删除大目录时应用无响应
- **修复建议**:
  - 改为 `tauri::async_runtime::spawn_blocking` 执行删除
  - 添加删除进度事件
  - 支持取消删除操作

### P1 - 应当修复

#### 4. 硬编码默认扫描路径
- **位置**: `src/components/Scanner.tsx:17`
- **问题**: 默认路径硬编码为 `/Users`
- **影响**: 首次使用需要手动选择目录
- **修复建议**: 
  - 使用 Tauri API 获取当前用户主目录
  - Rust 端: `dirs::home_dir()` 或 `std::env::var("HOME")`
  - 前端通过命令获取用户目录

#### 5. 缺少单元测试覆盖
- **位置**: 全局
- **问题**: 仅有一个 `is_sensitive_path` 测试，scanner.rs 核心逻辑无测试
- **影响**: 重构风险高，难以保障代码质量
- **修复建议**:
  - 添加 scanner.rs 单元测试：目录遍历、大小计算、排序逻辑
  - 添加 commands.rs 集成测试：扫描、取消、删除流程
  - 使用 `#[cfg(test)]` 模块组织测试代码
  - 考虑添加 property-based testing（proptest）

#### 6. 日志输出过于详细
- **位置**: `src-tauri/src/commands.rs:263`, `scanner.rs:100` 等
- **问题**: tracing 日志输出过于频繁，每条事件都打印
- **影响**: 生产环境日志文件过大
- **修复建议**:
  - 区分 debug/trace 级别
  - 生产环境只输出 info/warn/error
  - 添加日志轮转配置

#### 7. 错误处理可优化
- **问题**: 错误消息不够友好，如 `"扫描失败: ..."`
- **影响**: 用户难以理解错误原因
- **修复建议**:
  - 分类错误类型：权限问题、路径不存在、超时等
  - 提供具体解决建议
  - 前端展示友好错误提示

### P2 - 建议改进

#### 8. 无回收站机制
- **问题**: 删除是永久删除，无法恢复
- **建议**: 使用 `trash` crate 移动到 macOS 废纸篓

#### 9. 缺少设置持久化
- **问题**: 用户偏好（扫描目录、限制值）不保存
- **建议**: 
  - 使用 `tauri-plugin-store` 保存到本地 JSON
  - 或使用 Rust 端文件存储

#### 10. 无结果分页/虚拟滚动
- **问题**: 大量结果（>200）时一次性渲染有性能问题
- **建议**: 
  - FileList 组件使用 @tanstack/react-virtual
  - 或实现分页加载

## 🚀 可迭代功能

### P1 - 核心功能增强

#### 1. 结果导出
- **描述**: 支持导出扫描结果为 CSV、JSON
- **实现方案**:
  - Rust 端: serde_json / csv crate 生成文件
  - Tauri dialog 保存文件
  - 或前端直接生成 Blob 下载
- **价值**: 便于用户在其他工具中分析数据

#### 2. 高级排序与筛选
- **排序**: 按大小、名称、修改时间、类型（升序/降序）
- **筛选**: 按文件类型（图片/视频/文档等）、日期范围、大小范围
- **实现建议**: 
  - 后端实现排序/筛选逻辑（减少传输数据量）
  - 或前端实现（响应更快）
- **价值**: 精准定位目标文件

#### 3. 批量操作
- **描述**: 多选文件/目录，批量删除
- **实现建议**:
  - FileList 添加复选框列
  - 底部操作栏：全选、反选、删除选中
  - 确认对话框显示选中数量和总大小
- **价值**: 提高清理效率

---

### P2 - 可视化与体验

#### 4. 磁盘使用可视化
- **描述**: 饼图/环形图展示文件类型分布
- **实现方案**:
  - 使用 recharts 或 visx 库
  - 按文件扩展名分类统计
  - 可选：添加 Treemap 展示目录层级
- **价值**: 直观了解磁盘空间占用

#### 5. 扫描缓存
- **描述**: 保存扫描结果，避免重复扫描
- **实现方案**:
  - 使用 SQLite 存储扫描结果（rusqlite）
  - 按目录路径 + 修改时间作为缓存 key
  - 增量扫描：对比缓存，只处理变化的文件
- **价值**: 大幅提升重复扫描速度

#### 6. 文件预览
- **描述**: 快速预览文件内容
- **实现方案**:
  - 图片: base64 编码传输预览
  - 文本: 读取前 N 行
  - 使用 Tauri 的 fs plugin 读取文件
- **价值**: 确认文件内容后再删除

---

### P3 - 高级功能

#### 7. 重复文件检测
- **描述**: 查找内容相同的文件
- **实现方案**:
  - 第一步：按文件大小分组
  - 第二步：计算相同大小文件的 MD5/SHA256
  - 第三步：返回重复文件列表
- **价值**: 发现隐藏的磁盘空间浪费

#### 8. 定时扫描
- **描述**: 定期自动扫描指定目录
- **实现方案**:
  - macOS: 使用 launchd 或 cron
  - 或应用内定时器
  - 变化通知: 推送通知或邮件
- **价值**: 主动发现磁盘空间问题

#### 9. 目录对比
- **描述**: 对比两次扫描结果
- **显示**: 新增文件、删除文件、大小变化
- **价值**: 了解一段时间内的磁盘变化

#### 10. 主题切换
- **描述**: 深色/浅色主题，支持跟随系统
- **实现方案**:
  - Tailwind CSS darkMode: 'class'
  - 检测系统主题: `window.matchMedia('(prefers-color-scheme: dark)')`
  - 提供手动切换选项
- **价值**: 提升用户体验

#### 11. 多语言支持
- **描述**: i18n 国际化
- **实现方案**:
  - 使用 react-i18next
  - 支持中文、英文
  - 语言检测: 跟随系统或手动选择
- **价值**: 扩大用户群体
